# EdgeFlow C++ Runtime

Arduino/ESP32 library for running EdgeFlow ML models on-device with zero cloud dependency.

---

## Installation

### Option A — Arduino Library Manager (recommended)

Copy the `cpp/` folder into your Arduino libraries directory:

```
Windows : C:\Users\<you>\Documents\Arduino\libraries\EdgeFlow\
Mac/Linux: ~/Arduino/libraries/EdgeFlow/
```

Restart Arduino IDE. The library will appear under **Sketch > Include Library > EdgeFlow**.

### Option B — Manual (sketch folder)

Copy `src/` files directly into your sketch folder alongside your `.ino` file.

## Basic Usage

```cpp
// Step 1: Include generated model header FIRST
#include "safenest_model.h"    // auto-generated by: edgeflow export

// Step 2: Include EdgeFlow runtime
#include <EdgeFlow.h>

void setup() {
    Serial.begin(115200);
    EdgeFlow.begin();           // prints version banner to Serial
    EdgeFlow.printModelInfo();  // prints model type, features, classes
}

void loop() {
    // Step 3: Build input array (raw sensor values, same order as training CSV)
    float input[EDGEFLOW_N_FEATURES] = {
        (float)analogRead(34),   // MQ-2 gas sensor
        (float)digitalRead(27)   // PIR motion sensor
    };

    // Step 4: Run inference
    EdgeFlowResult result = EdgeFlow.predict(input, EDGEFLOW_N_FEATURES);

    // Step 5: Use result
    if (result.success) {
        Serial.print("Label:      "); Serial.println(result.label);
        Serial.print("Confidence: "); Serial.println(result.confidence);
        Serial.print("Latency ms: "); Serial.println(result.latency_ms);
    }

    delay(500);
}
```

---

## EdgeFlowResult Fields

| Field | Type | Description |
|-------|------|-------------|
| `label` | `int` | Predicted class index. Check your label map. |
| `confidence` | `float` | Confidence percentage (0–100) |
| `latency_ms` | `float` | Inference time in milliseconds |
| `memory_used_kb` | `float` | Heap used during inference in KB |
| `success` | `bool` | `true` if prediction succeeded |

## Supported Devices

| Device | Supported | Notes |
|--------|-----------|-------|
| ESP32 DevKit | **Yes** | Primary target — full timing and memory measurement |
| ESP8266 | Yes | Heap measurement supported |
| Arduino Uno/Mega (AVR) | Partial | No heap measurement; timing via `micros()` |
| Any Arduino-compatible | Yes | Timing works; memory returns 0.0 |

---

## Supported Models

Generated headers automatically select the correct inference engine at compile time via `EDGEFLOW_MODEL_TYPE`:

| `EDGEFLOW_MODEL_TYPE` | Model | C++ Runtime File |
|----------------------|-------|-----------------|
| `1` | Decision Tree | `models/DecisionTreeRuntime.h` |
| `0` | Logistic Regression | `models/LogisticRuntime.h` |
| `2` | Naive Bayes | `models/NaiveBayesRuntime.h` |

No manual selection required — include your model `.h` and the right engine is compiled in automatically.

## Library Structure

```
cpp/
├── library.properties          # Arduino library metadata
├── src/
│   ├── EdgeFlow.h              # Main header — include this in your sketch
│   ├── EdgeFlow.cpp            # Runtime implementation
│   ├── EdgeFlowResult.h        # Result struct
│   ├── models/
│   │   ├── DecisionTreeRuntime.h
│   │   ├── LogisticRuntime.h
│   │   └── NaiveBayesRuntime.h
│   └── utils/
│       ├── TimingUtils.h       # micros()-based latency
│       └── MemoryUtils.h       # ESP.getFreeHeap() memory tracking
└── examples/
    ├── BasicPredict/
    ├── BenchmarkDevice/
    └── SafeNestIntegration/
```

## Performance on ESP32

| Model | Avg Latency | RAM Used |
|-------|------------|---------|
| Decision Tree (2 features) | < 0.1 ms | ~0 KB heap |
| Logistic Regression (2 features) | < 0.5 ms | ~0 KB heap |
| Naive Bayes (2 features) | < 0.3 ms | ~0 KB heap |

All model weights are stored in Flash (PROGMEM) — RAM impact during inference is near zero.
